## Validation Rules

### Code Structure
All AC9 codes follow this pattern:
```
AC9E{y}{STRAND}{NUM}
```
Where:
- `AC9` = Australian Curriculum v9 prefix
- `E` = learning area letter (E, M, S, H, T, A, P, L)
- `{y}` = year level (F, 1, 2, 3, 4, 5, 6)
- `{STRAND}` = 2-letter strand code (LY, LR, LC, MA, ME, MP, MC, etc.)
- `{NUM}` = 2-digit number (01, 02, 03...)

### Quick Check Pattern
```
AC9 + [E|M|S|H|T|A|P|L] + [F|1|2|3|4|5|6] + [2-letter strand] + [01-10]
```

### INVALID Examples (do not use):
- AC9EMLY01 (wrong — Mathematics doesn't have LY)
- AC9E3SC99 (SC only goes to 08)
- AC9EFXYZ01 (XYZ is not a valid strand)
- AC9E7LY01 (Year 7 doesn't exist in F-6)
- AC9EFAR08 (AR only goes to 07)

### How to Validate a Code
1. Parse the learning area (letter 4)
2. Parse the year level (letter 5)
3. Parse the strand (letters 6-7)
4. Parse the number (last 2 digits)
5. Check if that exact combination exists in this file
6. If NOT in this file → CODE DOES NOT EXIST → reject it
